# Task 2 — Semantic HTML & Multi-Page Portfolio

A responsive, multi-page professional portfolio project focused on clean semantic HTML structural architecture, accessible form validation, explicit document style layouts, and structured block model mechanics.

## 📁 Repository Structure

* **index.html** — Core homepage introduction featuring structural hero blocks and clear navigation entry points.
* **about.html** — Full profile background tracking core technical studies, education history, and specialized learning timelines.
* **projects.html** — Modular portfolio grid layout showcasing structural project placeholders linked to descriptive vector assets.
* **contact.html** — Fully validated, accessible communication form built with field grouping structures and strict character validation limits.
* **semantic-conversion.html** — A dedicated refactoring exercise demonstrating the modernization of legacy, non-semantic layout structures into semantic tags.
* **box-model-practice.html** — Practical implementation workspace displaying detailed box model visualizations, margin constraint behavior, and border-box sizing calculations.
* **style.css** — Unified style sheet handling explicit container parameters, layout alignment, typography, interactive element pseudo-states, and global responsive breakpoints.
* **accessibility-report.md** — Dedicated evaluation document detailing core validation audits, contrast assessments, and structural accessibility compliance rules.

## 🛠️ Key Implementations Tracked

* **Semantic Content Architecture** — Completely avoided generic block wrappers in favor of clear layout headers, semantic main containers, and document section dividers.
* **Proportional Layout Mechanics** — Configured global layout dimensions, flexible box structural mechanics, and precise border-box tracking metrics.
* **Form Validation & Constraints** — Built input field parameters utilizing strict pattern definitions, mandatory selection constraints, and character minimum metrics.
* **Responsive Styling Framework** — Maintained consistent styling rules across all independent page documents using fluid structural blocks and unified class design definitions.
